# Agent 模块功能清单

## 模块概述

Agent 模块是 SDK 的核心执行层，负责管理 Agent Loop 的完整生命周期，包括实例创建、执行控制、状态管理、检查点机制等。该模块采用分层架构设计，各层职责清晰，支持同步执行、流式执行和异步执行三种模式。

## 架构层次

### 1. 实体层 (entities/)

**AgentLoopEntity**
- Agent Loop 执行实例的核心数据实体
- 封装执行实例的所有数据和状态访问接口
- 持有消息历史管理器和变量状态管理器
- 支持中止控制、中断控制和对话管理器集成
- 提供状态查询方法（运行中、暂停、完成、失败、取消）

**AgentLoopState**
- Agent Loop 执行状态管理器
- 管理执行状态转换（创建、运行、暂停、完成、失败、取消）
- 记录迭代历史和工具调用历史
- 跟踪执行时间（开始时间、结束时间）
- 支持中断标志管理（暂停、停止）
- 提供状态克隆功能

### 2. 管理器层 (execution/managers/)

**MessageHistoryManager**
- 管理消息历史记录
- 支持消息添加、获取、设置、清空操作
- 提供最近消息查询功能
- 支持消息历史规范化
- 提供快照创建和恢复功能

**VariableStateManager**
- 管理变量状态
- 支持变量的增删改查操作
- 提供批量变量获取功能
- 支持快照创建和恢复功能

### 3. 工厂层 (execution/factories/)

**AgentLoopFactory**
- 创建新的 AgentLoopEntity 实例
- 支持初始化消息历史和变量状态
- 支持设置对话管理器和父上下文（Thread ID、Node ID）
- 支持从检查点恢复 AgentLoopEntity 实例

### 4. 协调器层 (execution/coordinators/)

**AgentLoopCoordinator**
- 协调 AgentLoopEntity 的完整生命周期
- 支持同步执行、流式执行和异步执行三种模式
- 提供暂停、恢复、停止操作
- 管理执行实例的注册和注销
- 支持按状态查询实例（运行中、暂停、完成）
- 提供已完成实例的清理功能

**ConversationCoordinator**
- 协调对话级别的 Agent Loop 管理
- 管理多轮对话的上下文

### 5. 执行器层 (execution/executors/)

**AgentLoopExecutor**
- Agent Loop 的核心执行协调器
- 支持同步执行和流式执行
- 委托给 AgentIterationExecutor 处理单次迭代
- 委托给 AgentStreamExecutor 处理流式执行
- 支持中断信号检查（中止、暂停、停止）
- 集成事件管理器和错误处理

**AgentIterationExecutor**
- 执行单次 Agent Loop 迭代
- 调用 LLM 获取响应
- 处理工具调用
- 管理迭代状态和结果

**AgentStreamExecutor**
- 处理流式执行逻辑
- 流式转发 LLM 层事件
- 生成 Agent 层事件（工具调用、迭代完成等）
- 支持中断信号检查

### 6. 检查点层 (checkpoint/)

**AgentLoopCheckpointCoordinator**
- 协调完整的检查点流程
- 支持创建完整检查点和增量检查点
- 支持从检查点恢复 AgentLoopEntity
- 自动决定检查点类型（基于配置和检查点数量）
- 验证检查点完整性和兼容性

**AgentLoopDiffCalculator**
- 计算状态快照之间的差异
- 支持消息差异计算
- 支持变量差异计算
- 优化增量检查点存储

**AgentLoopDeltaRestorer**
- 从增量检查点恢复完整状态
- 处理增量链的递归恢复
- 支持降级到完整检查点

**AgentLoopCheckpointResolver**
- 解析检查点配置
- 管理检查点依赖项

### 7. 服务层 (services/)

**AgentLoopRegistry**
- 管理所有活跃的 AgentLoopEntity 实例
- 提供实例的注册、注销、查询功能
- 支持按状态过滤查询
- 提供实例清理功能

### 8. 处理器层 (execution/handlers/)

**AgentLoopLifecycle**
- 创建 Agent Loop 检查点
- 清理 Agent Loop 资源
- 克隆 Agent Loop 实体

**AgentErrorHandler**
- 处理 Agent Loop 执行过程中的错误
- 标准化错误为 SDKError
- 判断错误是否可恢复
- 处理中断错误（暂停、取消）
- 集成事件管理器触发错误事件

**HookHandler**
- 处理 Agent Loop 的钩子机制
- 支持执行前、执行后、错误处理等钩子
- 构建钩子上下文
- 触发钩子事件

## 核心功能

### 1. 执行控制

**同步执行**
- 执行 Agent Loop 并返回完整结果
- 支持最大迭代次数限制
- 支持工具调用和响应处理
- 自动处理执行状态转换

**流式执行**
- 流式返回执行事件
- 包含 LLM 层事件（文本增量、工具参数解析等）
- 包含 Agent 层事件（工具调用、迭代完成等）
- 支持实时中断控制

**异步执行**
- 异步启动 Agent Loop
- 立即返回实例 ID
- 后台执行不阻塞调用方
- 支持后续查询执行状态

### 2. 中断控制

**暂停功能**
- 设置暂停标志
- 在迭代边界检查暂停信号
- 暂停时保留当前状态
- 支持从暂停点恢复执行

**停止功能**
- 设置停止标志
- 触发 AbortController
- 立即取消执行
- 清理相关资源

**恢复功能**
- 重置中断标志
- 从暂停状态恢复执行
- 继续未完成的迭代

### 3. 状态管理

**执行状态**
- CREATED：已创建，未开始执行
- RUNNING：正在执行
- PAUSED：已暂停
- COMPLETED：执行完成
- FAILED：执行失败
- CANCELLED：已取消

**迭代记录**
- 记录每次迭代的开始和结束时间
- 记录每次迭代的工具调用
- 记录每次迭代的响应内容

**工具调用记录**
- 记录工具调用的 ID、名称、参数
- 记录工具调用的开始和结束时间
- 记录工具调用的结果或错误

### 4. 检查点机制

**完整检查点**
- 保存完整的状态快照
- 包含消息历史、变量状态、执行状态
- 作为增量检查点的基线

**增量检查点**
- 仅保存与上一个检查点的差异
- 减少存储空间占用
- 支持配置基线间隔和最大增量链长度

**检查点恢复**
- 从检查点恢复 AgentLoopEntity
- 支持完整检查点和增量检查点
- 自动处理增量链的递归恢复
- 验证检查点完整性

### 5. 错误处理

**错误标准化**
- 将原始错误标准化为 SDKError
- 构建完整的错误上下文
- 记录错误日志

**错误分级**
- ERROR：严重错误，停止执行
- WARNING：警告级别，可恢复
- INFO：信息级别，可恢复

**中断错误处理**
- 识别中止错误
- 区分暂停和取消
- 更新 Agent 状态

### 6. 事件机制

**事件类型**
- LLM 层事件：文本增量、工具参数解析等
- Agent 层事件：工具调用开始/结束、迭代完成等
- 自定义事件：用户定义的事件

**事件发射**
- 优先使用 EventManager
- 支持自定义事件发射函数
- 安全发射机制（捕获异常）

## 设计原则

### 1. 分层架构
- 实体层：纯数据实体，封装状态
- 管理器层：管理消息历史和变量状态
- 工厂层：集中管理实例创建
- 协调器层：编排复杂操作流程
- 执行器层：核心执行逻辑
- 检查点层：状态持久化
- 服务层：注册表等全局服务
- 处理器层：生命周期和错误处理

### 2. 无状态设计
- 协调器和执行器采用无状态设计
- 所有状态通过 AgentLoopEntity 管理
- 便于测试和扩展

### 3. 依赖注入
- 通过构造函数接收依赖
- 支持依赖替换和模拟
- 提高可测试性

### 4. 函数式导出
- 生命周期处理采用纯函数
- 与实体类解耦
- 便于复用和测试

### 5. 事件驱动
- 支持事件管理器集成
- 提供扩展点
- 支持异步事件处理
