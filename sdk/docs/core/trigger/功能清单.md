# Trigger 模块功能清单

## 一、核心类型定义

### 1. 基础类型（sdk/core/triggers/types.ts）
- **BaseTriggerCondition**: 触发条件基础接口，包含事件类型、自定义事件名称和元数据
- **BaseTriggerAction**: 触发动作基础接口，包含动作类型、参数和元数据
- **BaseTriggerDefinition**: 触发器定义基础接口，包含触发器ID、名称、描述、条件、动作、启用状态、最大触发次数等
- **TriggerExecutionResult**: 触发器执行结果，包含触发器ID、成功标志、执行动作、执行时间、结果数据和错误信息
- **TriggerStatus**: 触发器状态枚举（idle、active、triggered、disabled、expired）
- **BaseEventData**: 事件数据基础接口，包含事件类型、名称、数据、时间戳和来源ID
- **TriggerHandler**: 触发器处理器函数类型
- **TriggerMatcher**: 触发器匹配器函数类型

### 2. 工作流触发器类型（@modular-agent/types）
- **WorkflowTrigger**: 工作流触发器定义，扩展自基础触发器定义
- **TriggerReference**: 触发器引用，用于引用触发器模板
- **Trigger**: 完整的触发器对象，包含定义和运行时状态
- **TriggerRuntimeState**: 触发器运行时状态，包含触发器ID、线程ID、工作流ID、状态、触发次数等

## 二、核心功能模块

### 1. 触发器匹配器（sdk/core/triggers/matcher.ts）
- **defaultTriggerMatcher**: 默认触发器匹配器，匹配事件类型和事件名称
- **matchTriggerCondition**: 匹配触发条件，判断事件是否满足触发条件
- **matchTriggers**: 批量匹配触发条件，从触发器列表中找出所有匹配的触发器
- **createTriggerMatcher**: 创建自定义匹配器，支持自定义匹配逻辑

### 2. 触发器限制器（sdk/core/triggers/limiter.ts）
- **canTrigger**: 检查触发器是否可以触发，考虑启用状态和最大触发次数
- **getTriggerStatus**: 获取触发器状态（idle、active、triggered、disabled、expired）
- **incrementTriggerCount**: 增加触发计数
- **resetTriggerCount**: 重置触发计数
- **isTriggerExpired**: 检查触发器是否已过期
- **getRemainingTriggers**: 获取剩余触发次数

### 3. 触发器验证器（sdk/core/validation/trigger-validator.ts）
- **validateTriggerCondition**: 验证触发条件配置
- **validateTriggerAction**: 验证触发动作配置
- **validateWorkflowTrigger**: 验证工作流触发器配置
- **validateTriggerReference**: 验证触发器引用配置
- **validateTriggers**: 验证触发器数组，检查ID唯一性
- **validateExecuteTriggeredSubgraphActionConfig**: 验证触发子工作流动作配置
- **validateExecuteScriptActionConfig**: 验证执行脚本动作配置

### 4. 触发器模板注册表（sdk/core/services/trigger-template-registry.ts）
- **register**: 注册触发器模板
- **registerBatch**: 批量注册触发器模板
- **get**: 获取触发器模板
- **has**: 检查触发器模板是否存在
- **update**: 更新触发器模板
- **unregister**: 删除触发器模板
- **unregisterBatch**: 批量删除触发器模板
- **list**: 列出所有触发器模板
- **listSummaries**: 列出所有触发器模板摘要
- **clear**: 清空所有触发器模板
- **size**: 获取触发器模板数量
- **search**: 搜索触发器模板
- **export**: 导出触发器模板为JSON字符串
- **import**: 从JSON字符串导入触发器模板
- **convertToWorkflowTrigger**: 将触发器模板转换为工作流触发器

### 5. 预定义触发器（sdk/core/services/predefined-triggers.ts）
- **registerContextCompressionTrigger**: 注册上下文压缩触发器模板
- **registerContextCompressionWorkflow**: 注册上下文压缩工作流
- **registerContextCompression**: 同时注册上下文压缩触发器和工作流
- **unregisterContextCompressionTrigger**: 注销上下文压缩触发器
- **unregisterContextCompressionWorkflow**: 注销上下文压缩工作流
- **isContextCompressionTriggerRegistered**: 检查上下文压缩触发器是否已注册
- **isContextCompressionWorkflowRegistered**: 检查上下文压缩工作流是否已注册

### 6. 预定义资源定义 (sdk/resources/predefined/)
- **Workflow (workflow/context-compression.ts)**:
  - **createContextCompressionWorkflow**: 创建预定义的上下文压缩工作流
  - **createCustomContextCompressionWorkflow**: 创建自定义配置的上下文压缩工作流
- **Trigger (trigger/context-compression.ts)**:
  - **createContextCompressionTriggerTemplate**: 创建预定义的上下文压缩触发器模板
  - **createCustomContextCompressionTrigger**: 创建自定义配置的上下文压缩触发器模板

## 三、执行层组件

### 1. 触发器协调器（sdk/graph/execution/coordinators/trigger-coordinator.ts）
- **register**: 注册触发器，初始化运行时状态
- **unregister**: 注销触发器，删除运行时状态
- **enable**: 启用触发器
- **disable**: 禁用触发器
- **get**: 获取触发器（定义 + 状态）
- **getAll**: 获取所有触发器
- **handleEvent**: 处理事件，执行匹配的触发器
- **clear**: 清空所有触发器状态

### 2. 触发器状态管理器（sdk/graph/execution/managers/trigger-state-manager.ts）
- **register**: 注册触发器状态
- **getState**: 获取触发器状态
- **updateStatus**: 更新触发器状态
- **incrementTriggerCount**: 增加触发次数
- **createSnapshot**: 创建状态快照
- **restoreFromSnapshot**: 从快照恢复状态
- **getAllStates**: 获取所有状态
- **hasState**: 检查触发器状态是否存在
- **deleteState**: 删除触发器状态
- **cleanup**: 清理资源

### 3. 触发器处理器上下文工厂（sdk/graph/execution/factories/trigger-handler-context-factory.ts）
- **createHandlerContext**: 根据触发器动作类型创建对应的处理器上下文
- **hasCheckpointSupport**: 检查是否支持检查点功能
- **getThreadRegistry**: 获取线程注册表
- **getWorkflowRegistry**: 获取工作流注册表
- **getStateManager**: 获取触发器状态管理器
- **getCheckpointStateManager**: 获取检查点状态管理器
- **getGraphRegistry**: 获取图注册表
- **getEventManager**: 获取事件管理器
- **getThreadLifecycleCoordinator**: 获取线程生命周期协调器

### 4. 触发子工作流管理器（sdk/graph/services/triggered-subworkflow-manager.ts）
- **executeTriggeredSubgraph**: 执行触发子工作流
- **getTaskStatus**: 查询任务状态
- **cancelTask**: 取消任务
- **getQueueStats**: 获取队列统计信息
- **getPoolStats**: 获取线程池统计信息
- **getTaskStats**: 获取任务注册表统计信息
- **shutdown**: 关闭管理器
- **cleanupExpiredTasks**: 清理过期任务

## 四、触发器处理器

### 1. 处理器注册（sdk/graph/execution/handlers/trigger-handlers/index.ts）
- **getTriggerHandler**: 获取触发器处理函数
- **triggerHandlers**: 触发器处理函数映射表

### 2. 具体处理器实现
- **stopThreadHandler**: 停止线程处理器
- **pauseThreadHandler**: 暂停线程处理器
- **resumeThreadHandler**: 恢复线程处理器
- **skipNodeHandler**: 跳过节点处理器
- **setVariableHandler**: 设置变量处理器
- **sendNotificationHandler**: 发送通知处理器
- **customHandler**: 自定义动作处理器
- **executeScriptHandler**: 执行脚本处理器
- **executeTriggeredSubgraphHandler**: 执行触发子工作流处理器
- **applyMessageOperationHandler**: 应用消息操作处理器

## 五、触发器动作类型

支持的触发器动作类型：
1. **start_workflow**: 启动工作流（未实现）
2. **stop_workflow**: 停止工作流（未实现）
3. **stop_thread**: 停止线程
4. **pause_thread**: 暂停线程
5. **resume_thread**: 恢复线程
6. **skip_node**: 跳过节点
7. **set_variable**: 设置变量
8. **send_notification**: 发送通知
9. **custom**: 自定义动作
10. **execute_triggered_subgraph**: 执行触发子工作流
11. **execute_script**: 执行脚本
12. **apply_message_operation**: 应用消息操作

## 六、事件类型支持

触发器支持监听的事件类型包括：
- 线程生命周期事件：THREAD_STARTED、THREAD_COMPLETED、THREAD_FAILED、THREAD_PAUSED、THREAD_RESUMED、THREAD_CANCELLED等
- 节点执行事件：NODE_STARTED、NODE_COMPLETED、NODE_FAILED、NODE_CUSTOM_EVENT
- Token相关事件：TOKEN_LIMIT_EXCEEDED、TOKEN_USAGE_WARNING
- 消息事件：MESSAGE_ADDED
- 工具调用事件：TOOL_CALL_STARTED、TOOL_CALL_COMPLETED、TOOL_CALL_FAILED
- 检查点事件：CHECKPOINT_CREATED、CHECKPOINT_RESTORED、CHECKPOINT_DELETED、CHECKPOINT_FAILED
- 子图事件：SUBGRAPH_STARTED、SUBGRAPH_COMPLETED、TRIGGERED_SUBGRAPH_STARTED、TRIGGERED_SUBGRAPH_COMPLETED、TRIGGERED_SUBGRAPH_FAILED
- 变量事件：VARIABLE_CHANGED
- 用户交互事件：USER_INTERACTION_REQUESTED、USER_INTERACTION_RESPONDED等
- 错误事件：ERROR
- LLM流事件：LLM_STREAM_ABORTED、LLM_STREAM_ERROR
- 动态线程事件：DYNAMIC_THREAD_SUBMITTED、DYNAMIC_THREAD_COMPLETED等

## 七、设计原则

1. **类型安全**: 使用TypeScript类型系统和Zod验证确保配置正确性
2. **线程隔离**: 每个线程有独立的触发器状态管理
3. **状态分离**: 触发器定义与运行时状态分离管理
4. **依赖注入**: 通过工厂模式管理依赖，支持灵活配置
5. **可扩展性**: 支持自定义匹配器和处理器
6. **并发安全**: 状态管理器保证并发安全
7. **快照支持**: 支持状态快照和恢复，用于检查点机制
