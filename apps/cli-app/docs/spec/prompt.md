human relay如果通过tui实现，可能导致过大的渲染开销，也会导致必须实现更复杂的防抖、缓存。
需要尽可能保持cli应用的简单行。考虑是否应该把一些复杂输出全部通过文件io来实现。
例如tui内不再显示每个工具调用的具体信息，仅显示摘要，完整信息输出到特定文件(例如./wf-agent/output.txt)，human relay前输出到human-relay-output.txt中，用户则在human-relay-input.txt中输入信息
可以考虑是否应该在cli-app中提供统一的组件消息类型定义，方便控制所有类型的消息的输出模式，也可以让用户自己配置。
