# Frontend Design

This skill provides guidance for creating high-quality frontend interfaces.

## Design Principles

### 1. Visual Hierarchy

- Use size, color, and spacing to guide the user's eye
- Create clear focal points for key actions
- Balance visual weight across the layout

### 2. Typography

- Choose fonts that match the brand personality
- Establish a clear type scale (h1, h2, h3, body, small)
- Use line-height of 1.5-1.7 for body text
- Limit to 2-3 font families maximum

### 3. Color

- Start with a primary brand color
- Create a cohesive palette with 3-5 colors
- Ensure sufficient contrast for accessibility (WCAG AA minimum)
- Use color consistently for similar elements

### 4. Spacing

- Use consistent spacing scale (4px, 8px, 16px, 24px, 32px...)
- Group related elements with tighter spacing
- Separate distinct sections with generous whitespace
- Align elements on a grid

### 5. Interaction

- Provide clear visual feedback for interactive elements
- Use subtle animations for state changes
- Ensure touch targets are at least 44x44px
- Design for keyboard navigation

## Component Patterns

### Buttons

- Primary: High contrast, used for main actions
- Secondary: Lower contrast, used for alternative actions
- Tertiary: Text-only, used for less important actions

### Forms

- Group related fields logically
- Provide clear labels and helpful placeholder text
- Show validation errors inline
- Use appropriate input types (email, tel, number)

### Cards

- Use consistent padding and border radius
- Include clear visual hierarchy within cards
- Consider hover states for interactive cards

## Technology Stack

- React with TypeScript
- Tailwind CSS for styling
- Framer Motion for animations
- Lucide React for icons
