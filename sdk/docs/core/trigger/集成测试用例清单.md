# Trigger 模块集成测试用例清单

## 一、触发器匹配器测试（trigger-matcher.test.ts）

### 1. 基础匹配功能
- 测试事件类型匹配：相同事件类型应匹配成功
- 测试事件类型不匹配：不同事件类型应匹配失败
- 测试事件名称匹配：指定eventName时，事件名称应匹配
- 测试事件名称不匹配：指定eventName时，不同事件名称应匹配失败
- 测试未指定事件名称：未指定eventName时，只匹配事件类型

### 2. 批量匹配功能
- 测试批量匹配：从多个触发器中筛选出匹配的触发器
- 测试跳过禁用触发器：enabled为false的触发器应被跳过
- 测试自定义匹配器：使用自定义匹配逻辑进行匹配

### 3. 自定义匹配器
- 测试创建自定义匹配器：自定义匹配器应在默认匹配基础上执行自定义逻辑
- 测试自定义匹配器组合：自定义匹配器可以组合多个条件

## 二、触发器限制器测试（trigger-limiter.test.ts）

### 1. 触发能力检查
- 测试启用状态触发器：enabled为true或未设置时应可触发
- 测试禁用状态触发器：enabled为false时应不可触发
- 测试未达到最大次数：未达到maxTriggers时应可触发
- 测试达到最大次数：达到maxTriggers时应不可触发
- 测试无限制触发：maxTriggers为0或未设置时应可触发

### 2. 状态获取
- 测试空闲状态：未触发过的触发器状态应为idle
- 测试已触发状态：触发过的触发器状态应为triggered
- 测试禁用状态：enabled为false的触发器状态应为disabled
- 测试过期状态：达到maxTriggers的触发器状态应为expired

### 3. 计数管理
- 测试增加触发计数：触发计数应正确递增
- 测试重置触发计数：重置后计数应归零
- 测试剩余次数计算：正确计算剩余触发次数
- 测试无限制剩余次数：maxTriggers为0时应返回-1
