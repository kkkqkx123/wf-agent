# Agent 模块集成测试设计

## 测试策略

集成测试按照场景划分测试文件，每个测试文件专注于特定的功能场景，确保模块间协作的正确性。测试应覆盖正常流程、异常流程和边界条件。

## 测试文件划分

### 1. agent-loop-execution.test.ts - 执行流程测试

**测试场景：同步执行**
- 基本执行流程：创建实例、执行、返回结果
- 无工具调用的简单对话执行
- 包含工具调用的多轮迭代执行
- 达到最大迭代次数的执行
- 执行过程中 LLM 返回最终答案提前结束

**测试场景：流式执行**
- 流式执行基本流程
- 流式事件的正确顺序和类型
- LLM 层事件的正确转发
- Agent 层事件的正确生成
- 流式执行的中断处理

**测试场景：异步执行**
- 异步启动并立即返回实例 ID
- 后台执行完成后的状态更新
- 异步执行失败的状态更新

### 2. agent-loop-interruption.test.ts - 中断控制测试

**测试场景：暂停功能**
- 执行过程中暂停

**测试场景：恢复功能**

- 恢复后完成执行

**测试场景：停止功能**
- 执行过程中停止
- 停止后状态正确更新为 CANCELLED
- 停止后资源正确清理

**测试场景：中止控制**
- AbortController 的正确使用
- 中止信号的传播
- 中止错误的正确处理

## 测试数据准备

### Mock LLM Executor
- 模拟 LLM 调用和响应
- 支持配置不同的响应类型（文本、工具调用、混合）
- 支持模拟错误和延迟

### Mock Tool Service
- 模拟工具注册和执行
- 支持配置不同的工具行为
- 支持模拟工具错误

### Mock Event Manager
- 捕获所有触发的事件
- 支持事件验证和断言

### Mock Checkpoint Storage
- 模拟检查点的存储和检索
- 支持配置检查点数据
- 支持模拟存储错误

## 测试边界条件

### 迭代边界
- 零次迭代（立即返回）
- 一次迭代
- 最大迭代次数
- 超过最大迭代次数

### 消息边界
- 空消息历史
- 单条消息
- 大量消息
- 超长消息

### 工具边界
- 无工具
- 单个工具
- 多个工具
- 工具调用失败

### 状态边界
- 所有状态类型的转换
- 状态转换的非法操作
- 并发状态访问

### 检查点边界
- 第一个检查点
- 增量链的最大长度
- 检查点恢复失败
- 检查点数据损坏
