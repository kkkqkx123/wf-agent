# Graph 模块功能清单

## 模块概述

Graph 模块是 SDK 的核心工作流执行引擎，负责工作流的预处理、验证和执行。该模块基于图数据结构实现，支持复杂的流程控制（Fork/Join、子图、循环等）。

---

## 1. 实体层 (entities)

### 1.1 GraphData - 图数据结构
- **职责**: 存储图的节点、边及邻接关系
- **功能**:
  - 节点管理：添加、查询、存在性检查
  - 边管理：添加、查询、存在性检查
  - 邻接关系：正向/反向邻接表维护
  - 图遍历：获取出边/入边邻居、获取源点/汇点
  - 图查询：节点数量、边数量、所有ID列表

### 1.2 PreprocessedGraphData - 预处理后的图数据
- **职责**: 存储预处理完成后的完整图信息
- **功能**:
  - 继承 GraphData 的所有功能
  - ID 映射管理
  - 节点配置存储
  - 触发器配置存储
  - 子图关系管理
  - 图分析结果缓存
  - 拓扑排序结果
  - 子图合并日志

### 1.3 ExecutionState - 执行状态
- **职责**: 管理线程执行过程中的状态
- **功能**:
  - 检查点状态管理
  - 执行上下文维护
  - 节点执行计数

### 1.4 ThreadEntity - 线程实体
- **职责**: 封装 Thread 数据及其操作
- **功能**:
  - 线程状态管理（状态转换、当前节点）
  - 变量作用域管理（四级作用域）
  - 节点结果记录
  - 错误收集
  - 子图上下文管理（进入/退出子图）
  - Fork/Join 上下文管理
  - 事件构建
  - 对话管理器关联

---

## 2. 服务层 (services)

### 2.1 GraphRegistry - 图注册表
- **职责**: 管理预处理后的图
- **功能**:
  - 注册/注销图
  - 查询图
  - 批量操作
  - 获取工作流ID列表

### 2.2 WorkflowRegistry - 工作流注册器
- **职责**: 工作流定义的注册、查询和管理
- **功能**:
  - 工作流注册（自动触发预处理）
  - 工作流查询（按ID、名称、标签、分类、作者）
  - 工作流列表与搜索
  - 工作流引用关系管理
  - 工作流层次结构管理（父子关系）
  - 工作流导入/导出
  - 安全删除检查
  - 工作流验证
  - 活跃工作流管理

### 2.3 ThreadRegistry - 线程注册表
- **职责**: 管理线程实例
- **功能**:
  - 注册/注销线程
  - 查询线程
  - 线程状态追踪

### 2.4 ThreadPoolService - 线程池服务
- **职责**: 管理线程池
- **功能**:
  - 线程池配置
  - 线程复用
  - 资源限制管理

### 2.5 TriggeredSubworkflowManager - 触发子工作流管理器
- **职责**: 管理触发式子工作流
- **功能**:
  - 触发子工作流执行
  - 异步执行协调
  - 父子线程关联

---

## 3. 预处理层 (preprocessing)

### 3.1 GraphBuilder - 图构建器
- **职责**: 从工作流定义构建图
- **功能**:
  - 从 WorkflowDefinition 构建 GraphData
  - 节点和边的转换
  - Fork/Join Path ID 全局唯一化处理
  - 子工作流递归处理
  - 子工作流图合并

### 3.2 GraphNavigator - 图导航器
- **职责**: 提供图导航功能
- **功能**:
  - 获取起始节点
  - 获取下一个节点
  - 获取前一个节点
  - 获取邻居节点
  - 路径查找

### 3.3 WorkflowProcessor - 工作流预处理器
- **职责**: 工作流预处理流程
- **功能**:
  - 工作流验证
  - 节点引用展开
  - 触发器引用展开
  - 图构建与验证
  - 子工作流处理
  - ID 映射构建
  - 预处理图生成

### 3.4 IdMappingBuilder - ID映射构建器
- **职责**: 构建ID映射和配置
- **功能**:
  - 生成ID映射
  - 节点配置更新
  - 触发器配置更新
  - 子图关系映射

### 3.5 图工具 (utils)

#### 3.5.1 GraphAnalyzer - 图分析器
- **功能**:
  - 图结构分析
  - Fork/Join 配对收集
  - 完整分析报告

#### 3.5.2 GraphCycleDetector - 环检测器
- **功能**:
  - 检测图中是否存在环
  - 识别环中的节点和边

#### 3.5.3 GraphReachabilityAnalyzer - 可达性分析器
- **功能**:
  - 分析节点可达性
  - 识别不可达节点
  - 识别死胡同节点

#### 3.5.4 GraphTraversal - 图遍历
- **功能**:
  - DFS 遍历
  - BFS 遍历
  - 获取可达节点
  - 获取反向可达节点

#### 3.5.5 GraphTopologicalSorter - 拓扑排序器
- **功能**:
  - 计算拓扑排序
  - 处理有环图情况

---

## 4. 验证层 (validation)

### 4.1 GraphValidator - 图结构验证器
- **职责**: 验证图的拓扑结构和逻辑正确性
- **功能**:
  - START/END 节点入出度约束验证
  - 孤立节点检测
  - 环检测
  - 可达性分析
  - Fork/Join 配对验证
  - 子工作流存在性验证
  - 触发子工作流特殊约束验证

### 4.2 WorkflowValidator - 工作流验证器
- **职责**: 验证工作流定义的数据完整性
- **功能**:
  - 基本信息验证
  - 工作流类型与节点类型匹配验证
  - 节点验证（ID唯一性、配置schema）
  - 边验证（ID唯一性、引用完整性）
  - 配置验证
  - 触发器验证
  - 自引用检测
  - 工具配置验证

### 4.3 NodeValidator - 节点验证器
- **职责**: 节点配置验证
- **功能**:
  - 节点基本信息验证
  - 节点类型验证
  - 节点配置验证（委托给具体验证器）

### 4.4 节点类型验证 (node-validation)
针对每种节点类型的专门验证：

- **add-tool-validator**: ADD_TOOL 节点
- **context-processor-validator**: CONTEXT_PROCESSOR 节点
- **continue-from-trigger-validator**: CONTINUE_FROM_TRIGGER 节点
- **end-validator**: END 节点
- **fork-validator**: FORK 节点（路径配置验证）
- **join-validator**: JOIN 节点（路径匹配验证）
- **llm-validator**: LLM 节点（模型配置验证）
- **loop-end-validator**: LOOP_END 节点
- **loop-start-validator**: LOOP_START 节点
- **route-validator**: ROUTE 节点（条件配置验证）
- **script-validator**: SCRIPT 节点（脚本配置验证）
- **start-from-trigger-validator**: START_FROM_TRIGGER 节点
- **start-validator**: START 节点
- **subgraph-validator**: SUBGRAPH 节点（子工作流引用验证）
- **user-interaction-validator**: USER_INTERACTION 节点
- **variable-validator**: VARIABLE 节点

---

## 5. 执行层 (execution)

### 5.1 ThreadBuilder - 线程构建器
- **职责**: 从工作流创建 ThreadEntity
- **功能**:
  - 从预处理后的图构建线程
  - 模板缓存管理
  - 线程深拷贝
  - Fork 子线程创建
  - 变量初始化

### 5.2 ThreadExecutor - 线程执行器
- **职责**: 执行 ThreadEntity
- **功能**:
  - 执行线程
  - 验证工作流图存在性
  - 协调执行流程

### 5.3 协调器 (coordinators)

#### 5.3.1 ThreadExecutionCoordinator - 线程执行协调器
- **功能**:
  - 协调线程执行流程
  - 管理节点执行循环
  - 处理中断状态
  - 构建执行结果

#### 5.3.2 NodeExecutionCoordinator - 节点执行协调器
- **功能**:
  - 协调节点执行
  - 处理子图边界
  - 执行节点 Hooks
  - 检查点管理
  - 事件触发
  - 中断处理

#### 5.3.3 LLMExecutionCoordinator - LLM执行协调器
- **功能**:
  - 协调 LLM 调用
  - 流式响应处理
  - Token 使用追踪

#### 5.3.4 VariableCoordinator - 变量协调器
- **功能**:
  - 变量初始化
  - 变量作用域管理
  - 变量读写协调

#### 5.3.5 TriggerCoordinator - 触发器协调器
- **功能**:
  - 触发器执行协调
  - 触发器状态管理

#### 5.3.6 ThreadLifecycleCoordinator - 线程生命周期协调器
- **功能**:
  - 线程创建
  - 线程暂停/恢复
  - 线程停止
  - 线程完成

#### 5.3.7 ThreadOperationCoordinator - 线程操作协调器
- **功能**:
  - 线程操作编排
  - 批量操作协调

#### 5.3.8 CheckpointCoordinator - 检查点协调器
- **功能**:
  - 检查点创建协调
  - 检查点恢复协调

#### 5.3.9 ToolVisibilityCoordinator - 工具可见性协调器
- **功能**:
  - 工具可见性计算
  - 动态工具管理

### 5.4 处理器 (handlers)

#### 5.4.1 节点处理器 (node-handlers)
针对每种节点类型的执行处理器：

- **start-handler**: START 节点执行
- **end-handler**: END 节点执行
- **variable-handler**: VARIABLE 节点执行（变量赋值）
- **fork-handler**: FORK 节点执行（并行分支创建）
- **join-handler**: JOIN 节点执行（并行分支合并）
- **subgraph-handler**: SUBGRAPH 边界处理（进入/退出子图）
- **script-handler**: SCRIPT 节点执行（脚本执行）
- **llm-handler**: LLM 节点执行（大模型调用）
- **add-tool-handler**: ADD_TOOL 节点执行（动态添加工具）
- **user-interaction-handler**: USER_INTERACTION 节点执行（用户交互）
- **route-handler**: ROUTE 节点执行（条件路由）
- **context-processor-handler**: CONTEXT_PROCESSOR 节点执行（上下文处理）
- **loop-start-handler**: LOOP_START 节点执行（循环开始）
- **loop-end-handler**: LOOP_END 节点执行（循环结束）
- **start-from-trigger-handler**: START_FROM_TRIGGER 节点执行
- **continue-from-trigger-handler**: CONTINUE_FROM_TRIGGER 节点执行
- **agent-loop-handler**: Agent 循环执行

#### 5.4.2 触发器处理器 (trigger-handlers)
- **功能**: 各类触发器的执行处理

#### 5.4.3 Hook 处理器 (hook-handlers)
- **功能**: 节点生命周期 Hook 执行

#### 5.4.4 子图处理器
- **subgraph-handler**: 子图边界处理
- **triggered-subgraph-handler**: 触发式子图处理

### 5.5 管理器 (managers)

#### 5.5.1 VariableStateManager - 变量状态管理器
- **功能**: 变量状态维护和变更追踪

#### 5.5.2 CheckpointStateManager - 检查点状态管理器
- **功能**: 检查点状态维护

#### 5.5.3 ThreadLifecycleManager - 线程生命周期管理器
- **功能**: 线程状态机管理

#### 5.5.4 MessageHistoryManager - 消息历史管理器
- **功能**: 对话历史管理

#### 5.5.5 ToolContextManager - 工具上下文管理器
- **功能**: 工具上下文维护

#### 5.5.6 ToolVisibilityManager - 工具可见性管理器
- **功能**: 工具可见性计算和更新

#### 5.5.7 TriggerStateManager - 触发器状态管理器
- **功能**: 触发器状态维护

#### 5.5.8 TaskQueueManager - 任务队列管理器
- **功能**: 任务队列管理

#### 5.5.9 CallbackManager - 回调管理器
- **功能**: 回调注册和触发

#### 5.5.10 DynamicThreadManager - 动态线程管理器
- **功能**: 动态创建的线程管理

#### 5.5.11 InterruptionDetector - 中断检测器
- **功能**: 中断信号检测

#### 5.5.12 ThreadCascadeManager - 线程级联管理器
- **功能**: 父子线程级联操作

#### 5.5.13 TypeIndexManager - 类型索引管理器
- **功能**: 节点类型索引维护

## 6. 支持功能

### 6.1 事件系统
- 节点开始/完成/失败事件
- 子图开始/完成事件
- 线程暂停/取消事件
- 检查点创建事件

### 6.2 错误处理
- 执行错误处理
- 配置验证错误
- 运行时验证错误
- 错误上下文增强

### 6.3 变量系统
- 四级作用域：global、thread、local、loop
- 变量引用解析
- 变量变更追踪

### 6.4 检查点机制
- 检查点创建（前置/后置）
- 检查点恢复
- 中断检查点

### 6.5 工具系统
- 工具可见性计算
- 动态工具添加
- 工具上下文管理

### 6.6 子图机制
- 子工作流展开
- 子图边界处理
- 命名空间隔离
- 输入/输出映射

### 6.7 Fork/Join 机制
- 并行分支创建
- 分支同步
- 结果合并

### 6.8 触发子工作流
- 异步执行
- 父子线程关联
- 结果传递
